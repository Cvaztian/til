── 1.1 SISTEMAS NUMÉRICOS Y REPRESENTACIÓN ──

Conversiones Rápidas core

Bin→Hex: agrupar 4 bits desde la derecha. 1101 0011 → 0xD3

Hex→Bin: cada dígito hex → 4 bits. 0xA7 → 1010 0111

Bin→Dec: sumar potencias de 2 donde hay 1

BCD ≠ Binario: 47 BCD = 0100 0111, 47 bin = 00101111

Shifts = Multiplicación/División

x << n = x × 2ⁿ (más rápido que multiplicar) x >> n = x / 2ⁿ (⚠ signed: aritmético vs lógico)

Complemento a 2 entrevista

Obtención: invertir todos los bits + sumar 1

Ventaja: mismo circuito sumador para suma y resta

MSB = signo (0=+, 1=−), pero NO es signo-magnitud

⚠ Overflow: unsigned wrap-around está definido (módulo 2ⁿ). Signed overflow = undefined behavior en C. MISRA C prohíbe confiar en él.

Punto Flotante & Punto Fijo hw/sw

IEEE 754 (32-bit): [1 signo][8 exp][23 mantisa]

Valor = (-1)ˢ × 1.M × 2^(E-127)
⚠ NUNCA comparar floats con ==. Usar fabs(a-b) < ε. MISRA lo prohíbe.

Punto Fijo (alternativa sin FPU)

DeMorgan — el más importante

(A·B)' = A' + B' (A+B)' = A' · B' !(a && b) ↔ !a || !b !(a || b) ↔ !a && !b

NAND como Universal (4 transistores)

NOT: NAND(A,A) = A'

AND: NAND + NOT = 2 NANDs

OR: DeMorgan → 3 NANDs

En CMOS: NAND=4T, AND=6T. Herramientas de síntesis convierten todo a NAND.

Teoremas Clave

Identidad: A+0=A A·1=A Nulo: A+1=1 A·0=0 Complemento: A+A'=1 A·A'=0 Idempotencia: A+A=A A·A=A Involución: (A')'=A Absorción: A+AB=A A(A+B)=A Distributiva: A+BC=(A+B)(A+C) ←especial

SOP vs POS — Cuándo usar cada uno

Regla: Contá 1s y 0s en la tabla de verdad.
• Más 0s que 1s → SOP (menos mintérminos)
• Más 1s que 0s → POS (menos maxtérminos)
• Parejos → hacer K-map de ambos y comparar

SOP (Σ mintérminos): filas F=1, AND de variables, OR de términos → AND-OR / NAND-NAND

POS (Π maxtérminos): filas F=0, OR de variables negadas, AND de términos → OR-AND / NOR-NOR

K-Maps — Referencia Rápida core

Reglas de Agrupación

1. Orden Gray (00,01,11,10) — NO binario

2. Grupos: rectángulos de 1, 2, 4, 8, 16...

3. Envuelve bordes (toroide)

4. Grupos lo más grandes posible

5. Superponer OK — todo 1 cubierto al menos una vez

6. Don't cares (X): usar como 0 o 1 para agrandar grupos

7. Variables constantes en grupo → quedan. Variables que cambian → se eliminan

K-Map 4 Variables

CD: 00 01 11 10 AB=00 │ m0 │ m1 │ m3 │ m2 │ AB=01 │ m4 │ m5 │ m7 │ m6 │ AB=11 │m12 │m13 │m15 │m14 │ AB=10 │ m8 │ m9 │m11 │m10 │

Código Gray: consecutivos difieren en 1 bit. Evita glitches en transiciones.

Usos: K-maps, encoders rotativos, cruces de clock domain (FIFOs).

Bin→Gray: G[n]=B[n], G[i]=B[i+1]⊕B[i]

── 1.3 COMPUERTAS, TRANSISTORES Y NIVELES ──

CMOS — Transistores entrevista

NMOS

Conduce con Gate = 1

Pull-down (Source → GND)

Portadores: electrones

Movilidad: ~1350 cm²/V·s

Más rápido

PMOS

Conduce con Gate = 0

Pull-up (Source → VDD)

Portadores: huecos

Movilidad: ~480 cm²/V·s

Se hace 2-3× más ancho

NMOS más rápido porque: electrones tienen ~2.8× más movilidad que huecos en Si. Mayor movilidad → más corriente → carga capacitancias más rápido. Por eso NAND (NMOS en serie en pull-down) > NOR (PMOS en serie en pull-up).

Inversor CMOS

VDD │ ┌───┤ PMOS (conduce si In=0) │ │ In─┤ ├── Out │ │ └───┤ NMOS (conduce si In=1) │ GND

Consumo estático ≈ 0 (nunca ambos conducen simultáneamente).

Usos del inversor: NOT lógico, buffer/repeater (regenera señales degradadas en líneas largas), oscilador de anillo (# impar en loop), celda SRAM (2 inversores cruzados).

NAND CMOS — Anatomía

VDD / \ PMOS_A PMOS_B ← PARALELO (cualquier 0 → Out=1) \ / ├── Out │ NMOS_A ← SERIE (ambos 1 → Out=0) │ NMOS_B │ GND

4 transistores. AND necesitaría NAND+INV = 6T.

NAND es la compuerta natural y más eficiente en CMOS.

MUX — Función Universal entrevista

MUX 2:1 → Y = S'·I0 + S·I1 MUX 4:1 → Y = S1'S0'·I0 + S1'S0·I1 + S1S0'·I2 + S1S0·I3

MUX 2ⁿ:1 implementa cualquier función de N variables = tabla de verdad en hardware = LUT de FPGA.

Truco: N+1 vars con MUX 2ⁿ:1

Para F(A,B,C) con MUX 4:1 (selectores A,B): por cada combo AB, evaluar F vs C:

→ Embedded: MUX de alternate function en pines del micro (un pin puede ser GPIO, UART TX, SPI MOSI — el registro de AF configura el MUX interno).

Decodificador, Encoder, Sumadores

Decodificador N:2ⁿ

N bits → 2ⁿ salidas, solo una activa. Cada salida = AND de variables (comp. o no).

App: decodificación de direcciones de memoria (selecciona chip/periférico).

Priority Encoder

2ⁿ entradas → N salidas. Codifica la entrada de mayor prioridad.

App: controlador de interrupciones (NVIC en ARM).

Sumadores

Half: Sum=A⊕B Carry=A·B Full: Sum=A⊕B⊕Cin Cout=A·B+Cin·(A⊕B)

Ripple carry: N full adders en cadena. Simple, lento (carry propaga en serie).

Carry lookahead: G=A·B, P=A⊕B. Carries en paralelo. Rápido, más HW.

ALU = sumador + control → AND, OR, XOR, +, −, CMP.

Comparador (Flags)

Internamente hace A−B, evalúa flags:

N (Negative)MSB del resultado
V (Overflow)Overflow signed
A==B: Z=1 A>B(uns): C=1,Z=0 A
── 1.5 CIRCUITOS SECUENCIALES ──

Latch vs Flip-Flop entrevista

Latch D

Enable=1 → Q sigue a D (transparente)

Enable=0 → Q mantiene

Problema: D cambia, Q cambia

Flip-Flop D (Master-Slave)

Solo captura en el borde del clock

Clk=0: master acepta D, slave bloqueado

Rising edge: master bloquea, slave abre

Clk=1: D puede cambiar, Q inmune

Setup & Hold Time

D estable ANTES del borde ≥ t_setup D estable DESPUÉS del borde ≥ t_hold Violar → METAESTABILIDAD

¿Por qué D domina sobre JK?

D: 1 entrada, Q=D en borde. Simple, verificable, herramientas optimizadas para D.

JK: 2 entradas, 4 modos (hold/set/reset/toggle). Era útil en diseño manual. Hoy irrelevante — los FFs físicos en FPGA/ASIC son tipo D.

Flip-Flop T

T=1 → toggle en cada borde = divisor de frecuencia ÷2 (no es oscilador libre — necesita clock externo).

T=0 → hold. Base de contadores binarios.

Metaestabilidad & Sincronización hw

No se "limpia" — se mitiga con sincronizador:

Señal async → [FF1] → [FF2] → Señal sync │ │ Clock Clock

FF1 puede metaestabilizar, pero tiene T_clock para resolverse. P(fallo) = e^(−T/τ) ≈ ridículamente bajo.

Triple FF para aviónica: 3 ciclos latencia, probabilidad aún menor.

⚠ Regla: toda señal que cruza clock domain o viene del exterior → sincronizador de ≥2 FF. En aviónica no es opcional.

Contadores

Async (ripple): clock solo al 1er FF. Simple pero glitches (bits no cambian juntos). Solo si velocidad no importa.

Sync: mismo clock a todos. Sin glitches. Úsalo siempre. Apps: timer, PWM, watchdog, PC.

FSM — Máquinas de Estado entrevista

Moore vs Mealy

Moore: salida = f(estado). Más predecible. Cambia 1 ciclo después.

Mealy: salida = f(estado, entrada). Más rápida pero posibles glitches.

Diseño Síncrono

T_clk > t_prop(FF) + t_delay(comb) + t_setup(FF)

Violación = timing violation = datos incorrectos.

Clock domains: sincronizadores o FIFOs Gray entre dominios.

Recovery States — Obligatorio en aviónica

1. Estado ERROR explícito en el diagrama

2. default: en switch → ir a ERROR

3. Timeout por estado (watchdog) → ERROR si excede

4. One-hot: detectar estados inválidos con lógica

5. DO-178C: análisis exhaustivo de TODAS las transiciones

switch(state) { case IDLE: ... break; case RUN: ... break; case ERROR: disable_outputs(); log_error(ERR_FSM); state = IDLE; break; default: state = ERROR; break; // ← nunca omitir }

volatile — Pregunta #1 en entrevistas

Qué hace: prohíbe al compilador cachear/optimizar accesos. Cada lectura/escritura en código → lectura/escritura real a memoria.

Cuándo: registros HW, variables compartidas con ISR, variables RTOS, registros read-to-clear.

Qué NO hace: no es atómico (ISR puede interrumpir entre bytes), no garantiza ordenamiento completo (usar memory barriers).

── TEMAS AVANZADOS (SELLO MIT/TEC) ──

Hazard Glitches core

Pulso espurio en la salida durante transición de entrada, causado por delays desiguales en caminos internos.

Static-1 hazard: salida debería ser 1 pero baja momentáneamente a 0.

Static-0 hazard: salida debería ser 0 pero sube momentáneamente a 1.

Ejemplo y Solución

F = AB + A'C (hazard cuando A: 1→0, B=C=1) Fix: F = AB + A'C + BC (término de consenso)

En K-map: asegurar que todos los pares de 1s adyacentes estén cubiertos por al menos un grupo, aunque el grupo sea redundante.

⚠ Peligroso si: la salida combinacional alimenta un clock, enable o reset asíncrono. En lógica que alimenta D de un FF síncrono, generalmente no causa problemas (FF muestrea en el borde, no durante el glitch).

Detección y Corrección de Errores hw

MétodoDetectaCorrigeUso
Paridad1 bit—ARINC 429, buses, UART

Paridad: 1 bit extra → total de 1s par/impar. XOR de todos los bits.

Hamming: bits de paridad en posiciones 2ⁿ. Cubren subconjuntos específicos. Síndrome = cuáles fallaron → indica bit erróneo.

CRC: división de polinomios en GF(2) (aritmética XOR). Implementado con LFSR.

→ Aviónica: memorias con ECC obligatorias. ARINC 429 usa paridad impar. CRC en protocolos de datos.

Aritmética HW Adicional

Restador con Sumador

A − B = A + NOT(B) + 1 Cada bit de B pasa por XOR con señal SUB: SUB=0 → B directo (suma) SUB=1 → B invertido + Cin=1 (resta)

Un solo sumador + XOR por bit + MUX = suma Y resta.

Multiplicación Binaria

1101 (13) × 1011 (11) ------ 1101 (AND con bit 0, shift 0) 1101 (AND con bit 1, shift 1) 0000 (AND con bit 2, shift 2) 1101 (AND con bit 3, shift 3) -------- 10001111 (143) → N×N bits = 2N resultado

HW: arreglo de ANDs + árbol de sumadores (Wallace/Dadda tree).

Micros 8-bit sin multiplicador HW → software (shifts+sumas), lento.

Tristate Buffers y Buses entrevista

Tristate Buffer

Enable=1 → Output = Data Enable=0 → Output = Hi-Z (desconectado) Enable │ Data ──[▷── Output (0, 1, o Hi-Z)

Buses compartidos: múltiples dispositivos en el mismo bus. Solo 1 habla a la vez, el resto en Hi-Z.

Contención: dos dispositivos activos simultáneamente = cortocircuito → daño. Chip select (del decodificador) controla quién habla.

Open-Drain / Open-Collector

Salida solo puede tirar a GND (activa) o flotar (inactiva). Pull-up resistor externo compartido.

Wired-AND: cualquier dispositivo puede tirar la línea a bajo.

I2C: SDA y SCL son open-drain. Múltiples masters posibles. Clock stretching (slave mantiene SCL bajo para pedir tiempo).

Interrupciones: múltiples fuentes comparten una línea IRQ con open-drain.

── ERRORES COMUNES ──

⚠ Trampas Frecuentes

1. Confundir BCD con binario puro

2. Signed overflow = UB en C (no confiar en wrap)

3. Usar int en vez de uint32_t en embedded

4. Comparar floats con ==

5. Orden binario en K-maps (debe ser Gray)

6. Olvidar que K-map envuelve bordes

7. Conectar 5V a pin 3.3V sin level shifter

8. Confundir latch (nivel) con flip-flop (borde)

9. Olvidar volatile en registros HW

10. No sincronizar señales entre clock domains

11. FSM sin default / estado de recovery

12. Asumir que lectura desgasta Flash (solo escritura)

13. No agregar término de consenso para eliminar hazards

14. Dos dispositivos activos en bus → contención

15. Olvidar paridad/ECC en buses de aviónica

16. Multiplicación N×N sin reservar 2N bits para resultado
