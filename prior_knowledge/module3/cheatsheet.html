3.1a UART Core
Tipo: Asíncrono, 2 cables (TX/RX cruzados), punto a punto, full-duplex
Frame 8N1: Start(1) + Data(8) + Stop(1) = 10 bits → eficiencia 80%
Baud rates clave: 9600 (GPS) · 115200 (debug) — tolerancia ±3-5%
Throughput 115200 baud: 115200 / 10 = 11520 B/s ≈ 11.25 KB/s
Capas: UART = protocolo · TTL = niveles eléctricos · RS-232 = capa física invertida · RS-485 = diferencial
⚠️ TX→RX cruzado. TX con TX = error #1. Baud rate diferente = basura.
3.1b SPI EntrevistaCore
Tipo: Síncrono (SCLK), 3+N cables, full-duplex, 1 master / N slaves
Líneas: SCLK · MOSI (COPI) · MISO (CIPO) · CS̄ por slave (activo LOW)
Mecanismo: Shift register circular — 8 pulsos SCLK intercambian 1 byte entre master y slave
Siempre full-duplex: Enviar = recibir. Solo leer → enviar dummy. Solo escribir → ignorar recibido.
Velocidad: Hasta ~50+ MHz — el bus más rápido de los 4

3.1c I2C EntrevistaCore
Tipo: Síncrono (SCL), 2 cables (SCL+SDA), half-duplex, open-drain + pull-up
Pull-ups: 4.7 kΩ típico a 3.3V. Fast mode → 2.2 kΩ o 1 kΩ.
Topología: Multi-master, multi-slave. Hasta 127 dispositivos (7-bit address).
Wired-AND: Cualquier LOW gana → arbitraje natural + clock stretching.
Protocolo: START → Addr(7)+R/W̄(1) → ACK → Data → ACK → STOP
Lectura: 2 fases — write (dir registro) + repeated start + read (dato)
Velocidades: 100 kHz (standard) · 400 kHz (fast, más usado) · 1 MHz (Fm+) · 3.4 MHz (Hs)
Colisión de address: Pines A0/A1/A2 o I2C multiplexer (TCA9548A)
Uso: Sensores lentos, EEPROM, RTC, PMICs, IO expanders
⚠️ START/STOP se definen por SDA cambiando mientras SCL está HIGH. Datos solo cambian con SCL LOW.
3.1d CAN Bus ⭐ EntrevistaAviónicaCore
Tipo: Asíncrono, 2 cables diferenciales (CAN_H/CAN_L), multi-master (peers), half-duplex
Terminación: 120Ω en cada extremo del bus
Dominante (0): V_diff ≈ 2V (CAN_H=3.5V, CAN_L=1.5V) — transceiver separa líneas
Recesivo (1): V_diff ≈ 0V (ambas ≈ 2.5V) — estado default
Regla clave: Dominante SIEMPRE gana → base del arbitraje
Arbitraje: ID menor = mayor prioridad. Nodos comparan bit a bit; quien envía 1 pero lee 0 se retira. Sin colisión ni pérdida.
Frame: SOF + ID(11b) + RTR + IDE + DLC(4b) + Data(0-8B) + CRC(15b) + ACK + EOF
CAN FD: Hasta 64 bytes payload, 8 Mbps en datos (arbitraje sigue a 1 Mbps)
5 mecanismos de error: Bit monitoring · Bit stuffing · CRC · ACK · Frame check
Error confinement: Error Active (0-127) → Error Passive (128-255) → Bus Off (>255) — auto-aislamiento
Velocidades: 1 Mbps/40m · 500kbps/100m · 125kbps/500m · 50kbps/1000m
Aviación: ARINC 825 (CAN) · ARINC 429 (legacy, unidireccional) · AFDX (Ethernet)
💡 En entrevista: "¿protocolo de comunicación en aviones?" → ARINC 825 (CAN-based, moderno).
⚠️ IDs bajos = alta prioridad. Mensajes safety-critical siempre con IDs bajos.

3.2a GPIO FirmwareCore
Registros clave: MODER (modo) · OTYPER (PP/OD) · OSPEEDR · PUPDR · IDR (input) · ODR (output) · BSRR (atómico)
MODER: 00=Input · 01=Output · 10=AF · 11=Analog — 2 bits por pin
BSRR: Bits [15:0] = SET, bits [31:16] = RESET. Atómico — sin read-modify-write.
Alternate Function: AF0-AF15 en GPIOx_AFR[0/1]. Definidas en datasheet por pin.
⚠️ Error #1: Olvidar habilitar clock RCC antes de configurar GPIO. Sin clock → registros leen 0.
3.2b Interrupciones / NVIC EntrevistaCore
NVIC: Nested Vectored Interrupt Controller. Número menor = mayor prioridad.
Flujo: IRQ → HW push (R0-R3,R12,LR,PC,xPSR) → Vector table → ISR → HW pop → Resume
Latencia: 12 ciclos (Cortex-M4). Push/pop automático en hardware.
Tail-chaining: ISR pendiente → salta directo sin pop+push — solo 6 ciclos.
Late arrival: IRQ mayor prioridad durante push → cambia vector en vuelo.
⚠️ Reglas ISR (pregunta de entrevista):
1. Corta — setear flag, procesar en main/RTOS
2. Sin delays, bloqueantes, printf, malloc
3. Siempre limpiar pending flag
4. Variables compartidas → volatile
5. Secciones críticas → atómico o deshabilitar IRQs brevemente
3.2c Timers y PWM EntrevistaFirmware
Parámetros: PSC (prescaler) · ARR (auto-reload) · CNT (counter) · CCR (compare, para PWM)
Frecuencia overflow: f = f_clk / ((PSC+1) × (ARR+1))
PWM duty: Duty = CCR / (ARR+1) × 100% — CNT < CCR → HIGH, CNT ≥ CCR → LOW
Ejemplo 500ms (84MHz): PSC=8399 → 10kHz · ARR=4999 → T=0.5s
Ejemplo 1µs resolución: PSC=83 → 1MHz · 1 tick = 1µs
Ejemplo servo 50Hz: PSC=83, ARR=19999 → 50Hz · CCR=1500 → 1.5ms (centro)
⚠️ Trampa APB: Si prescaler APB > 1, clock del timer se multiplica ×2.
3.2d DMA CoreFirmware
Qué es: Transferencia HW sin CPU: periférico ↔ memoria o memoria ↔ memoria.
Modos: Normal (para al terminar) · Circular (loop continuo — ADC, audio)
Double buffering: DMA llena A mientras CPU procesa B. Cero muestras perdidas.
Bus stalling: DMA y CPU comparten bus AHB — DMA constante puede ralentizar CPU.
💡 Cuándo usar: Polling (raro/simple) · Interrupciones (frecuente, poco dato) · DMA (masivo, CPU libre)

3.4 Timing y Sincronización CoreFirmware
Clock sources: HSE (cristal, preciso) · HSI (RC, rápido) · LSE (32768 Hz = 2¹⁵ → RTC) · LSI (watchdog)
PLL: f_out = (HSE / M) × N / P — esperar PLL_RDY antes de usar
Ejemplo STM32F4: 8MHz / 8 × 336 / 2 = 168 MHz SYSCLK
Bus tree: SYSCLK → AHB (/1) → APB2 (/2, rápido) / APB1 (/4, lento)
Jitter: Variación de flancos. Afecta ENOB del ADC y timing serial.
CDC multi-bit: Gray code (contadores) · Handshake · Async FIFO
⚠️ En aviónica, TODO CDC debe ser documentado y verificado. CDC no sincronizado = bug latente.
3.5 JTAG / SWD / Debug Core
JTAG: 4 cables (TCK, TMS, TDI, TDO). Daisy-chain. Boundary scan + debug + programming.
SWD: 2 cables (SWDIO, SWCLK). Solo ARM. Debug equivalente, sin boundary scan.
Capacidades: Breakpoints (HW 4-8 / SW ∞) · Watchpoints · Single-step · Memory R/W · ITM trace
BIST: Auto-test HW al encender. Firmware invoca y maneja resultados (pass/fail).
💡 SWD para debug diario de Cortex-M (menos pines). JTAG para PCBs con múltiples chips o FPGAs.
